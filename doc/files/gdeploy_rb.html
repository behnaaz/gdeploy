gDeploy

Description

gDeploy est un petit script écrit en ruby, dans le but de déployer 

 et de configurer les services de base du midelware de grilles de
 calcul gLite.

gDeploy utilise les classes ruby net/ssh,scp. L‘environnement utilisé pour le déploiement est un système Scientific Linux 5.5, et la version de gLite est la 3.2

Public Instance methods

clusters(nodes)

Autres clusters dans la VO et dans la réservation passée.

conf_batch(batch, cehost, sname)

Configuration de la brique batch, composé d‘un server torque, et de maui. <QUEUE_NAME>_GROUP_ENABLE=<list of vo> Les users/groups sont configurés plus loin.

conf_bdii(bdii, sname, cehost)

Configuration de la brique Bdii, Site-Bdii, annuaire de la Vo. sname correspond au nom de la vo, nous utiliserons le nom du site.

conf_cehost(sname, cehost, batch, se)

Computing element, surcouche au batch server intègre une couche proxy, nous utilisons cream.

conf_groups(sname)

Groupes de la vo. "/<vo>":::: "/<vo>/<group>":::: "/<vo>/<group>/ROLE=<role>:::: "/<vo>/<group>/ROLE=<role>:::<special_user_type>:

conf_users(group,sname)

Utilisateurs pour la vo de test 3 admin et 9 utilisateurs, en fonction du nom de la vo. <user_id>:<username>:<group_id>:<group_name>:<vo_name>:<special_user_type>:

conf_wn(bdii, se, sname, batch, cehost)

Configuration des workers sous scientificlinux 5.5 La partie de déclaration de vo est très important elle détermine la vo/certif des noeuds.
